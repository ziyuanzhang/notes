# 数据规范 -> 数据模型 -> 数据结构 -> 数据

```bash
                    数据规范（Schema）
            （定义"应该长什么样"，不存数据）
    ┌────────────────────────────────────────────┐
    │ JSON Schema                                │
    │ OpenAPI Schema                             │
    │ XML Schema (XSD)                           │
    │ Protocol Buffers (.proto)                  │
    │ Avro Schema                                │
    │ GraphQL Schema                             │
    └────────────────────────────────────────────┘
                        │
                        │ 定义
                        ▼
                    数据模型（Model）
        （Python 中的数据对象，有字段、有行为）
    ┌────────────────────────────────────────────┐
    │ dataclass                                  │
    │ Pydantic BaseModel                         │
    │ attrs                                      │
    │ NamedTuple                                 │
    │ SQLAlchemy ORM Model                       │
    └────────────────────────────────────────────┘
                        │
                        │ 保存
                        ▼
                    数据结构（Data Structure）
        （真正存储数据的 Python 对象）
    ┌────────────────────────────────────────────┐
    │ dict                                       │
    │ list                                       │
    │ tuple                                      │
    │ set                                        │
    └────────────────────────────────────────────┘
                        │
                        │ 序列化 / 反序列化
                        ▼
                数据交换格式（Serialization）
    ┌────────────────────────────────────────────┐
    │ JSON                                       │
    │ XML                                        │
    │ YAML                                       │
    │ MessagePack                                │
    │ Protocol Buffers Binary                    │
    └────────────────────────────────────────────┘
```

Schema 定义 Model，Model 保存 Data，Data 最终序列化成 JSON。

1. 第一类：数据(Data)

   dict、list、tuple、set 这些都是：Data Structure（数据结构）

2. 第二类：Model（数据模型）：

   Model 是：把 Data 封装成一个对象。
   Model 的特点：✅ 有字段；✅ 有类型；✅ 可以有方法
   Model = Data + 行为 + 类型

3. 第三类：Schema（规范）

   Schema：永远记住一句话：Schema 不保存数据，只描述


```bash
      Database
           │
           ▼

           │
           ▼

           │
           ▼
         dict
           │
    json.dumps()
           │
           ▼

```

```bash
JSON（传输格式）
        │
        ▼
dict（Python 原始数据）
        │
        ├────────► dataclass（轻量数据模型）
        │
        └────────► Pydantic Model（带校验的数据模型）
                             │
                             ▼
                    JSON Schema（自动生成的数据规范）
```

- 在 FastAPI 的实际开发中，最常见的组合就是：
  - JSON：HTTP 请求和响应的数据格式。
  - dict：Python 内部的基础数据结构，也是很多库之间交换数据的“通用语言”。
  - Pydantic v2 (BaseModel)：定义请求体、响应体和配置对象，负责校验、转换和序列化。
  - JSON Schema：由 Pydantic 自动生成，供 OpenAPI/Swagger 使用，从而生成接口文档和客户端代码。

## dataclass 与 Pydantic

- dataclass 更常用于业务领域模型、配置对象或不需要运行时校验的内部数据，在 FastAPI 项目中通常与 Pydantic 配合使用，而不是完全替代 Pydantic。

  一句话先说明：`dataclass 更适合"程序内部的数据"，Pydantic 更适合"程序外部的数据"`。

  ```bash
  外部（HTTP、JSON、数据库、Redis...）
          │
          ▼
     Pydantic 负责校验
          │
          ▼
  ---------------------------
         业务逻辑开始
  ---------------------------
          │
          ▼
     dataclass 表示业务对象
  ```

  1. 例1：业务领域模型（最典型）
  2. 例2：配置对象（Config）
  3. 例3：函数之间传递数据
  4. 例4：算法计算结果

  `obj['key']`没有 `obj.key`好用；`obj['key']`key容易写错 和 不知道还有哪些key

## dict与dataclass 、Schema 与 Pydantic

1.

- 字典(dict)、json：
  1. JSON 字符串（JSON String）: 用于网络传输、文件存储的文本格式。 类型：str
     - 示例：`'{"name":"Tom","age":18}'`

  1. JSON 对象（JSON Object）: 是 JSON 规范中的一种数据类型，表示由键值对组成的数据。
     - 示例（JSON）：`{"name": "Tom", "age": 18}`

  1. Python dict（字典）: JSON Object 在 Python 中通常映射为 dict。
     - 例如：`json.loads('{"name":"Tom","age":18}') --> {'name': 'Tom', 'age': 18} --> 类型：dict`

- Schema（数据规范）、JSON Schema（JSON 数据规范）、JSON（数据）
  1. Schema: 是数据规范（Specification），用于描述数据应该满足什么结构和约束。
  2. JSON Schema: 是一种使用 JSON 表示的 Schema，用于描述 JSON 数据结构，并可用于：
     - 数据校验（Validation）
     - 自动生成接口文档
     - 自动生成客户端/服务端代码
     - IDE 自动补全

  ```JSON Schema 长什么样
     {
       "type": "object",
       "properties": {
         "name": {
           "type": "string",
           "minLength": 1
         },
         "age": {
           "type": "integer",
           "minimum": 0
         }
       },
       "required": ["name", "age"]
     }
  ```

  3. JSON: 是一种数据交换格式（Data Interchange Format），用于表示和传输数据。

- 字典(dict)、dataclass、pydantic2、JSON Schema、json
  1. dict: 是一个具体的数据容器，没有字段约束。
  2. dataclass(数据类): 是一种数据类型（模板），可以创建很多具有相同结构的数据对象，但不负责验证数据。
  3. Pydantic: 是 Python 中基于"类型注解"实现"数据模型(Model)、数据校验(Validation)、数据转换(Parsing)、序列化(Serialization)，并能够自动生成 Schema(JSON Schema)" 的库。
     Pydantic = dataclass + 类型校验 + 数据转换 + 序列化

- 公式
  1. JSON: JSON = 数据交换格式
  2. dict: dict = 数据容器
  3. dataclass: dataclass = 轻量数据模型
  4. Pydantic: Pydantic=数据模型+类型校验+类型转换+序列化+JSON Schema生成
  5. Schema: Schema = 数据规范
  6. Model: Model = Schema + Data + Behavior(行为)
     也就是：Model = 字段+类型+方法+对象

## 数据类型 与 数据类

- 数据类型

```bash
object
 ├── int, float, bool, complex(复数)
 ├── str(不可变)
 ├── list, tuple(不可变), range
 ├── set, frozenset(不可变集合)
 ├── dict
 ├── bytes(不可变字节), bytearray, memoryview
 ├── None(空值)
 └── function, class, ...

```

- 数据类

| 类型                 | 是否官方 | 是否推荐  | 特点                |
| -------------------- | -------- | --------- | ------------------- |
| `@dataclass`         | ✅       | ✅ 强烈   | 现代 Python 标准 ✅ |
| `namedtuple`         | ✅       | ⚠️        | 老方案，不直观      |
| `typing.NamedTuple`  | ✅       | ⚠️        | 不可变              |
| `attrs`              | ❌       | ✅        | dataclass 超集      |
| `pydantic.BaseModel` | ❌       | ✅（Web） | 校验 + 序列化 ✅    |

```bash
🔹 容器层（装数据）
   tuple / dict / list / collections

🔹 轻量结构体（结构 + 可读性）
   namedtuple / typing.NamedTuple / dataclass

🔹 增强结构体（更多控制）
   attrs

🔹 数据模型层（校验 + 序列化 + 工程）
   pydantic.BaseModel
```

🧭 你的“选择流程”

- 1️⃣ 普通业务数据: 用 @dataclass（默认答案）
- 2️⃣ API、JSON、外部输入: 用 pydantic
- 3️⃣ 性能 + 不可变: 用 namedtuple
- 4️⃣ 觉得 dataclass 不够用: 再考虑 attrs

👉 dataclass / pydantic 决定“一个数据长什么样”；
👉 collections 决定“一堆数据怎么高效地玩”；

```python
# 1. 默认
@dataclass
class Point:
    x: int
    y: int
    z: int = 18 # 默认值
    members: list[str] = field(default_factory=list) # 使用 field() 的数据类
    members: list[str] = []  # 大坑 ❌
p = Point(1, 2)
p.x = 10   # ✅ 可以改

# 2. 不可变数据类
@dataclass(frozen=True)
class Point:
    x: int
    y: int
p = Point(1, 2)
p.x = 10  # ❌ 报错

# 3. 默认值
@dataclass
class Team:
    name: str
    age: int = 18 # 默认值
    members: list[str] = field(default_factory=list) # 使用 field() 的数据类
    members: list[str] = []  # 大坑 ❌
    classVal: str = field(compare=False) # 不参与比较

# 4. 排序数据类
@dataclass(order=True)
class User:
    age: int
    name: str
User(18, "A") < User(20, "B")  # True

```
