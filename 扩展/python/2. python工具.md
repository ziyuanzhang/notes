- uv 与 uvicorn
  1. uv 是负责管理项目依赖和环境的“包管理器”，
  2. uvicorn 是负责运行 Web 应用的“服务器”。

## 管理工具

| 功能         | pip              | uv                |
| ------------ | ---------------- | ----------------- |
| 安装 Python  | ❌               | ✅                |
| 创建虚拟环境 | venv             | ✅                |
| 安装依赖     | pip install      | `uv add`          |
| 锁定依赖     | requirements.txt | `uv.lock`         |
| 同步环境     | 手动             | `uv sync`         |
| 运行命令     | 激活环境后运行   | `uv run`          |
| 下载速度     | 一般             | 很快（Rust 实现） |

| ------- | ---------------- | ------------------------------ | ----------------- | ---------- | ------------- | ------------ |
| 🥇 NO.1 | uv               | 全家桶（环境 + 包管理 + 构建） | 🚀 未来标准       | 100∼180MB  |    1.2∼2.8s   |  <45s        |
| 🥈 NO.2 | venv + pip-tools | 稳定组合                       | 🧱 企业保守首选   |   90∼150MB |     1.8∼3.5s  |    <90s      |

- 👉 一句话：uv = Python 世界的 npm + cargo；
- 👉 conda 是“重型武器”，不是日常工具（AI / 数据科学）；
  - conda 解决的是 系统级依赖（C库）；
  - pip/uv 解决不了 CUDA / MKL / BLAS；

## 安装 → 创建项目 → 管理 Python → 管理依赖 → 开发 → 运行 → 团队协作

最佳实践流程（推荐）

```bash
   安装 uv
      │
      ▼
uv init my_project
      │
      ▼
uv python install 3.12
      │
      ▼
uv python pin 3.12
      │
      ▼
   uv venv
      │
      ▼
调整为 src 布局（推荐）
      │
      ▼
uv add requests
uv add --dev pytest mypy ruff
      │
      ▼
配置 pyproject.toml
      │
      ▼
  uv sync
      │
      ▼
开发代码（src/）
      │
      ▼
uv run ruff check .
uv run ruff format .
uv run mypy src
uv run pytest
      │
      ▼
git add pyproject.toml
git add uv.lock
```

### 一、uv 安装

1. 安装
   - macOS / Linux: `curl -LsSf https://astral.sh/uv/install.sh | sh`
   - Windows (PowerShell): `powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"`
   - 备选安装方式
     - macOS (Homebrew): `brew install uv`
     - Windows (Winget): `winget install uv`
     - 通过 pipx 隔离安装: `pipx install uv`

2. 验证安装：`uv --version`

### 二、创建项目

1. 初始化新项目：`uv init my_project`; `cd my_project`

```BASH
   my_project
   ├── src/
   │   └── my_project/
   │       ├── __init__.py
   │       └── main.py
   ├── tests/
   ├── pyproject.toml
   ├── uv.lock
   └── .python-version
```

推荐：正式项目优先采用 src 布局，可避免导入路径污染。

### 三、管理 Python 版本

1. 安装指定 Python 版本：`uv python install 3.12`
2. 锁定项目 Python 版本：`uv python pin 3.12`（会生成或更新 .python-version 文件）

   这样 IDE、uv 都知道当前项目使用哪个 Python。

3. 查看 Python `uv python list`

### 四、创建虚拟环境

1. `uv venv` 会自动创建：`.venv/`;
2. 也可以指定 Python：`uv venv --python 3.12`
3. 激活虚拟环境（可选）
   - Linux/macOS: `source .venv/bin/activate`
   - Windows (PowerShell/CMD): `.venv\Scripts\activate`

**注意:** 使用 uv run 时，一般不需要手动激活虚拟环境。

### 五、管理依赖

1. 安装依赖: `uv add requests`自动更新：pyproject.toml 和 uv.lock 文件。
2. 安装开发依赖（如测试工具）：`uv add --dev pytest mypy ruff`
3. 删除依赖：`uv remove requests`
4. 查看依赖：`uv list` 显示依赖树。

### 六、同步环境

当团队成员拉取项目后：`uv sync`

- uv 会自动：
  1. 创建虚拟环境
  2. 安装依赖
  3. 根据 uv.lock 保持一致版本

  所以一般不需要： pip install -r requirements.txt

### 七、运行项目（推荐）

1. 推荐：（最省心的命令）: `uv run python main.py`
2. 或者：`uv run pytest`, `uv run mypy src`, `uv run ruff check .`

- 使用 uv run 是最推荐的运行方式。它会自动在项目的虚拟环境中执行代码，完全不需要你手动去 `source .venv/bin/activate` 激活环境：

### 八、项目配置

1. 推荐安装：`uv add --dev pytest mypy ruff`
2. 然后在 pyproject.toml 添加：

```bash
   [tool.mypy]
   python_version = "3.12" #项目锁定
   strict = true
   # 项目结构
   packages = ["my_project"]
   mypy_path = ["src"]
   # 常见妥协
   ignore_missing_imports = true
   warn_unused_ignores = true
   warn_return_any = true
   warn_unreachable = true

   [tool.ruff] # 格式 + lint
   target-version = "py312" #项目锁定
   line-length = 88

   [tool.ruff.lint] # 格式 + lint
   select = ["E", "F", "I", "B", "UP"]
   ignore = ["E501"]
```

### 九、代码检查

1. Lint：`uv run ruff check .`
   - 自动修复：`uv run ruff check . --fix`

2. 代码格式化：`uv run ruff format .`
3. 类型检查：`uv run mypy src`
4. 测试：`uv run pytest`

### 十、Git 配置

提交：`git add pyproject.toml , git add uv.lock`
不要提交：`.venv/`

```bash
   # Virtual Environment
   .venv/

   # Python Cache
   __pycache__/
   *.py[cod]

   # Tool Cache
   .mypy_cache/
   .pytest_cache/
   .ruff_cache/

   # Environment Variables
   .env

   # IDE
   .idea/
   .vscode/
```
