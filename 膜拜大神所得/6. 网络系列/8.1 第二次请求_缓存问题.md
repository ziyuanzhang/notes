# 8.1 第二次请求 - 缓存问题

## 浏览器 HTTP 缓存主要是按 URL（资源地址）存的，不是 IP

浏览器检查缓存时，核心依据：URL

包括：协议 + 域名 + 端口 + 路径 + 查询参数

即：https://example.com:443/js/app.js?v=1

## 第二次点击按钮时

真正“决定要不要发请求”的：是浏览器， 准确说：浏览器网络模块（network stack）在检查缓存。

- 浏览器内部大概结构

  ```bash
    JS代码
      ↓
    fetch/axios
      ↓
    浏览器 Web API
      ↓
    浏览器网络模块
      ↓
    缓存检查
      ↓
    决定是否联网
  ```

- 第二次请求时 DNS/TCP 还会发生吗？: 不一定。
  1. DNS Cache缓存: "域名 → IP" 所以通常：不需要重新 DNS 解析。
  2. TCP Keep-Alive: 复用 TCP 连接。所以通常：不需要重新三次握手。
  3. TLS Session Resume: HTTPS 可恢复 TLS 会话。所以通常：不需要完整 TLS 握手。

- 浏览器缓存读取优先级（现代）

  ```bash
    Service Worker
        ↓
    Memory Cache
        ↓
    Disk Cache
        ↓
    协商缓存（304）
        ↓
    网络请求（200）
  ```

- Service Worker（高级缓存）: 浏览器后台独立线程

  可：
  - 拦截请求
  - 离线缓存
  - PWA
  - 自定义缓存策略

  甚至: 断网也能运行页面。

## 第二次请求，整体流程（核心）

```bash
              用户再次请求资源
                  ↓
              浏览器检查缓存
                  ↓
              是否命中强缓存？
   ┌───────────────┴───────────────┐
   是                              否
   ↓                               ↓
直接使用缓存                    发起协商缓存请求
 （不联网）                          ↓
                           服务器检查资源是否变化
                                   ↓
                      ┌────────────┴────────────┐
                    未变化                     已变化
                      ↓                          ↓
                   返回304                    返回200+新资源
                      ↓                          ↓
                 浏览器使用本地缓存           更新缓存并使用新资源
```

### 第一层：强缓存（最重要）

强缓存：浏览器直接使用本地缓存，不发网络请求。

浏览器怎么知道能不能直接用？靠响应头;

```bash
   # 1. Cache-Control（现代主流）
   Cache-Control: max-age=3600 # 表示：3600秒内直接用本地缓存
                : public # 允许：浏览器/CDN/代理服务器缓存
                : private # 只允许：浏览器缓存
                : no-cache # 可以缓存，但每次必须问服务器资源是否更新(❗即：必须走协商缓存)。
                : no-store # 表示：完全不缓存
                : must-revalidate # 表示：缓存过期后，必须重新验证,不能直接使用旧资源。

   # 2. Expires（老方案）
   Expires: Wed, 21 Oct 2026 07:28:00 GMT # 表示：这个时间前 都直接使用缓存

```

### memory cache（内存缓存） 🆚 disk cache（磁盘缓存）

- memory cache（内存缓存）：资源存在：浏览器进程占用的电脑内存（RAM）里。

  可能因：“关闭 tab、内存压力、浏览器策略” 而释放；

  常见缓存内容：小图片、JS、CSS、高频资源

- disk cache（磁盘缓存）：资源存在：硬盘。

  浏览器关闭后可能还在；速度比内存慢一点；生命周期长。

### 第二层：协商缓存（重要）

如果强缓存过期：浏览器不能直接使用缓存，但：不一定重新下载资源。而是：问服务器：“资源变了吗？”

服务端第一次返回：`ETag: "abc123"`或`Last-Modified: Tue, 01 May 2026 12:00:00 GMT`浏览器记住。

第二次请求时,浏览器会带上：

```bash
   # 1. ETag方案（主流）: 当前资源唯一标识（更准确，通常：文件hash、版本号）
   If-None-Match: "abc123" # 表示：我本地版本是 abc123, 你看看变了没？

   # 2. Last-Modified方案: 资源最后修改时间（精度是秒：1秒内修改两次可能检测不到）
   If-Modified-Since: Tue, 01 May 2026 12:00:00 GMT # 表示：这个时间之后资源改过吗？

```

- 如果：资源没变, 返回：304 Not Modified, 📌注意：没有响应体

  因为：浏览器本地已经有资源, 浏览器：直接使用本地缓存

- 如果资源变了, 服务器返回：200 OK, 并返回：新资源, 浏览器更新缓存。

### 为什么协商缓存还要发请求？

因为：浏览器不确定资源有没有更新，只能问服务器。
