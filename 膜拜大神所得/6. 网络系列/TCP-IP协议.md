# TCP/IP

## 对比

1. 分层模型（统一理解）

   ```code
   OSI（七层模型教学）  TCP/IP （五层实现）  工程理解（六层推荐你用）
   --------------------------------------------------------
   应用(7)            应用层              应用层
   表示(6)
   会话(5)
   --------------------------------------------------------
   传输(4)            传输层              传输层
   --------------------------------------------------------
   网络(3)            网络层              寻址层 + 转发层
   --------------------------------------------------------
   链路(2)            数据链路层          标识层
   --------------------------------------------------------
   物理(1)            物理层              设备层
   ```

   设备层 → 标识层 → 寻址层 → 转发层 → 传输层 → 应用层

2. 每一层“只做一件事”（最重要理解）

   | 层     | 解决问题   | 核心概念   |
   | ------ | ---------- | ---------- |
   | 应用层 | 干什么     | HTTP / DNS |
   | 传输层 | 怎么可靠传 | TCP / UDP  |
   | 寻址层 | 目标是谁   | IP         |
   | 转发层 | 怎么过去   | 路由       |
   | 标识层 | 下一跳是谁 | MAC        |
   | 设备层 | 怎么发     | 网卡/信号  |

3. 数据是怎么一步步封装的

   ```code
      应用层：  data
       ↓
      传输层：  TCP + data  → Segment（报文段）
       ↓
      网络层：  IP + Segment → Packet（包）
       ↓
      链路层：  MAC + Packet → Frame（帧）
       ↓
      物理层：  bit → 电信号
   ```

4. 网络三大核心机制（必须掌握）
   - 1️⃣ 寻址（IP）： 决定：目标是谁（最终目的地）
     → 提供源IP、目标IP

   - 2️⃣ 转发（路由）： 决定：下一跳是谁
     1. 查路由表（最长前缀匹配）
     2. 判断：
        - 同一网段 → 直接交付
        - 不同网段 → 交给网关（默认路由）

   - 3️⃣ 交付（MAC）： 决定：把数据交给谁
     - 如果不知道MAC：使用ARP：IP → MAC
     - 封装二层帧：目标MAC = 下一跳设备的MAC

   - 👉 精华总结：IP定终点，路由定路径，MAC做交付。  
      IP → 路由 → ARP → MAC

   - 👉 重要特性：
     - 每一跳都会重新封装 MAC（因为下一跳变了）
     - IP 一般不变（除非发生 NAT）

5. 局域网 vs 外网（本质）  
   🟢 局域网: ARP → 找MAC → 直接发  
   🔴 外网: 发给网关 → 路由 → NAT → 互联网
6. 默认路由链路

   ```code
     你的电脑
       ↓（DHCP给默认网关）
     你家路由器
       ↓（ISP给默认路由）
     运营商网络
       ↓（BGP全球路由）
     互联网
       ↓
     目标服务器
   ```

7. 网络本质 🔥网络只干三件事：
   1. 找人（DNS / IP）
   2. 找路（路由 / 网关）
   3. 送达（MAC / TCP）

## 详情

### OSI 七层模型--开放式系统互联模型 ( 只是概念模型 -- 没有实际用 )

1. 应用层 ： 网络主机提供**方法和接口**，直接对应用户行为；例如 HTTP、FTP、SMTP；

2. 表示层（语法层） ：对数据 序列化，加密解密，压缩与解压；

3. 会话层：管理会话；例如 Open/Close/ReOpen/检查状态等；

4. 传输层：主机到主机的数据通信；
   - 建立连接保证数据封包发送、接受顺序一致；
   - 提供可靠性（发送者知道数据有没有完成送达）；
   - 提供流控制；
   - 提供多路复用

5. 网络层：提供数据在逻辑单元（IP 地址）之间的传递；
   - 路由：决定数据下一站在哪
   - 寻址：为数据封包增加头信息（地址等）

6. 数据链路层：提供数据在设备和设备间传输（网卡->路由器->路由器->防火墙->等等）；
   - 流控制：发送者和接收者之间同步数据的 收发速度和数据量；
   - 错误流程：可以检测错误（CRC），❌ 一般不负责重传（以太网不做）
     👉 重传主要在 TCP（传输层）

7. 物理层：定义底层一个个位（bit）的数据怎么转化为物理信号；例如：二进制怎么转化电压高低，如何转化为其他物理介质上的信号；
   - 将数据传递行为转化为物理设备识别的信号；

**注**

- 应用层 ------|
- 表示层 ------|-->用户数据
- 会话层 ------|
- 传输层 --------->数据封包
- 网络层 --------->帧、帧、帧
- 数据链路层------>二进制
- 物理层 --------->物理信号

### TCP/IP 协议模型（工程用）

1. 应用层 ----> http/FTP/SMTP等
   - 功能: 应用间通信，定义数据格式
   - 数据形态: 原始数据（根据协议规定的数据格式，程序员自己填充的数据）

2. 传输层 ----> TCP/UDP 协议（主机到主机）
   - 提供端口号（例:80）；
   - UDP: 只管发送（无连接、不可靠、不保证顺序，但开销小、延迟低）；
   - TCP: 负责 分段（Segment）、排序（序列号）、重传（丢包补发）、流量控制、拥塞控制；
   - 数据结构:
     1. TCP Header: 为原始数据 增加头部信息；包括：源端口/目标端口，序列号/确认号，标志位（SYN/ACK/FIN）
     2. TCP Data: 应用层数据转换后的数据单元；
   - 数据单元: 报文段（Segment），此时数据被切割成适合传输的大小（Segment = TCP Header + 应用层数据）。

3. 网络层 ----> IP 协议（源IP地址‘从哪来’ --> 目标IP地址‘到哪去’）
   - 数据结构:
     1. IP Header: 为传输层的数据 再增加头部, 包括源IP地址和目标IP地址；
     2. IP 数据: 把传输层数据封装成 IP 包（Packet）；
   - 数据单元: Packet（数据包）

   `网关`: 网关 = 默认出口（下一跳）；通常是路由器的一个接口IP;  
   👉 网关本质: 离开当前网络的“默认出口”

4. 链路层 ----> 多种底层网络协议，例: Ethernet（以太网）, wifi(设备-->设备), ⚠️ ARP协议(根据IP找MAC)；
   - ARP协议: 只在局域网内生效; 当只知道目标 IP，但不知道 MAC 时，通过 ARP 查询;
   - 作用: 同一局域网内: 设备 → 设备;
   - 数据结构:
     1. Frame Header: 为每个 IP 包添加头部【当前MAC地址、下一跳MAC地址】;
     2. Frame Data:
     3. Frame Footer: 为每个 IP 包添加尾部【差错检测（CRC）】;
   - 数据单元: 数据帧（Frame）;

   关联设备: 网卡（MAC 地址、以太网等）  
   每经过一个路由器: IP 不变, MAC 重写

5. 物理层 ----> 传输介质: 双绞线、同轴电缆、光纤
   - 功能: 将二进制数据（0/1）转换为电信号或光信号，进行物理传输
   - 数据单元: 比特（Bit）

### IPV4与IPV6 协议

#### IPV4 协议: 4 字节（32 位二进制）

1. 采用 4 个字节（32 位）来表示 IP 地址;
2. 1 个字节(8 位)：`2*2*2*2*2*2*2*2=2^8=256`; 取值范围: 0 ~ 255;
3. IPv4 协议理论上可以提供约 42 亿个独立的地址 `256*256*256*256 = 256^4 = (2^8)^4 = 2^32 =4,294,967,296`;

#### IPV6 协议

`2001:0db8:85a3:0000:0000:8a2e:0370:7334`

1. 每个十六进制字符：用 4 位二进制 表示（因为 `2**4=16`,刚好对应 0-9 和 a-f）。
2. 每组 4 个字符：4 字符 ×4 位/字符=16 位。
3. 总共 8 组：8 组 ×16 位/组=128 位。
4. `2**128 大约 3.4×10**38`

## ip地址与子网掩码地址

| 名字               | 地址              | 转义                                 |
| :----------------- | :---------------- | :----------------------------------- |
| IP地址             | 172.16.10.1       | 10101100.00010000.00001010.000000001 |
| 子网掩码           | 255.255 255.255.0 | 11111111.11111111.11111111.000000000 |
| （合成的）网络地址 | 172.16.10.0       | 10101100.00010000.00001010.000000000 |

- 确定可用主机范围：在这个子网中，所有 IP 地址的前三位都必须是 172.16.10。最后一位用来区分不同的设备，范围从 1 到 254（0 是网络地址，255 是广播地址，都不能用作主机地址）
  1. 同一个局域网：直接给目标电脑；
  2. 不同局域网：发给网关；

- 广播域：广播域是一个逻辑上的网络范围。在这个范围内，任何一个设备发出的广播信息，都会被域内的所有其他设备接收到。

- 通过路由器或虚拟局域网（VLAN）等技术将一个大的网络分割成多个更小的广播域，可以有效限制广播信息的传播范围，减少不必要的流量。

## 网络

```code
  设备层 → 标识层 → 寻址层 → 转发层 → 传输层 → 应用层
  对应现实：
  网卡  →  MAC  →   IP  →  路由 → TCP/UDP → HTTP/DNS
```

### 基础硬件层（设备）

1. 网卡（NIC）: 电脑连接网络的“接口”

   每个网卡都有：
   - IP地址（逻辑）
   - MAC地址（物理）

2. 交换机（Switch）: 局域网内“帮你找到人”
   - 根据 MAC 地址转发数据
   - 不看 IP

3. 路由器（Router）: 帮你“出门找路”
   - 根据 IP 地址决定转发路径
   - 连接不同网络

4. 调制解调器（Modem / 光猫）: 帮你接入 ISP（运营商）

### 地址与身份（标识）

1. MAC地址: 网卡的“身份证”（物理地址）
   - 示例：AA-BB-CC-DD-EE-FF
   - 局域网通信靠它

2. IP地址: 网络世界的“门牌号”, 用来定位设备
   - 👉 用来在网络中唯一标识一台设备
   - IPv4：192.168.1.1
   - IPv6：更长

3. 端口（Port）: 一台机器里的“应用编号”
   - IP = 找到机器
   - 端口 = 找到程序

### 网络划分（范围）

1. 子网掩码（Subnet Mask）: 判断是不是“同一个局域网”

   ```code
   IP:       192.168.1.10
   子网掩码:  255.255.255.0

   👉 表示：
      前 3 段（192.168.1）是“网络号”
      最后一段是“主机号”

   👉同一网段：192.168.1.xxx  → 局域网
   ```

2. 网关（Gateway）: 出局域网的“门”
   - 电脑的“网口”
   - 每个网卡都有一个：IP地址、MAC地址（物理地址）

     但凡接入互联网的主机必须有一块网卡，每块网卡在出厂时都烧制好一个全世界独一无二的地址（mac地址）；

   - IP决定“去哪”
   - MAC决定“怎么送到下一跳”

3. 广播地址: “通知所有人”的地址

   👉 作用：
   - ARP广播（找别人MAC地址）
   - 局域网发现设备

4. 广播域: 广播能到达的范围
   - 广播域 = 二层（MAC层）范围

   特点：
   - 局域网内是一个广播域
   - 路由器会隔离广播

### 核心协议（规则）

1. ARP（Address Resolution Protocol）: IP → MAC 转换
   - 问：谁是 192.168.1.1？
   - 答：我是，我的MAC是XX

2. IP协议: 负责“把数据送到目标IP”
   - 不保证可靠
   - 只负责“尽力送”

3. ICMP: 网络“诊断工具”
   - ping 就是它

4. TCP: 可靠传输（打电话）

   特点：
   - 三次握手
   - 有确认机制
   - 不丢数据

5. UDP: 不可靠但快（发短信）
   - 不保证到达
   - 适合视频/游戏

### 路由与转发（核心）

1. 路由表: 决定“下一跳去哪”
2. 默认路由: 不认识的都走这里
   - 0.0.0.0/0

3. NAT: 私网IP → 公网IP
   - 家庭上网必备

4. BGP: 互联网“导航系统”
   - ISP 各大网络交换路由信息

### 一个请求的完整路径（把所有概念串起来）

👉 访问 google.com 时：

1. DNS → 获取IP
2. 判断是否同网段
3. ARP → 找网关MAC
4. 发给网关
5. 路由器查路由表
6. NAT转换公网IP
7. 经过互联网（BGP）
8. 到达服务器
9. TCP建立连接
10. HTTP请求数据

### 最核心的5个理解

1. IP vs MAC
   - IP：决定去哪
   - MAC：决定下一跳怎么送

2. 局域网 vs 互联网
   - 局域网：ARP + MAC通信
   - 外网：靠路由器

3. 默认网关: 不认识的都交给它
4. 路由本质: 每一跳只知道“下一步”
5. 分层思想（最重要）
   - 应用层：你要干嘛（HTTP）
   - 传输层：怎么传（TCP）
   - 网络层：往哪走（IP）
   - 链路层：怎么发（MAC）

### 网络本质就三件事

1. 找到人（IP / DNS）
2. 找到路（路由 / 网关）
3. 把数据送过去（MAC / TCP）

### 👉 网络通信 = 这几件事叠加

- 应用：你要干嘛（HTTP）
- 传输：怎么保证到达（TCP）
- 寻址：目标是谁（IP）
- 转发：怎么走过去（路由）
- 标识：下一跳是谁（MAC）
- 设备：怎么发出去（网卡）

## 🟢 场景1：访问“局域网内”设备

例子

```code
你：192.168.1.10
目标：192.168.1.20
```

- step：① 判断是不是同一网段

  用子网掩码计算：

  ```code
  192.168.1.10 → 192.168.1
  192.168.1.20 → 192.168.1
  ✔ 相同 → 局域网内
  ```

- step：② 查MAC地址（关键！）

  电脑需要知道对方的MAC地址

  👉 通过 ARP（地址解析协议）

  发送：`广播：谁是 192.168.1.20？`

  → 局域网所有设备都会收到（广播域）

  → 只有目标回应：`我是，我的MAC是 XX-XX`

- step：③ 直接通信

  之后：`IP包 → 封装成 → MAC帧`

  发送给：`目标MAC地址`

  👉 不经过网关 ❗

- ✅ 总结一句话：👉 局域网内通信 = ARP找MAC → 直接发

## 🔴 场景2：访问“局域网外”（比如百度）

例子

```code
你：192.168.1.10
目标：8.8.8.8（公网）
```

- step: ① 判断网段

  ```code
  192.168.1.xxx ≠ 8.8.8.8
  ❌ 不在同一局域网
  ```

- step: ② 发给“网关”

  电脑会这样想：👉 “我不认识这个地址，交给网关处理”

- step: ③ 找网关MAC（ARP）
  `广播：谁是 192.168.1.1？`

  网关回复：`我是，我的MAC是 YY-YY`

- step: ④ 数据发给网关

  数据结构：

  ```code
    源IP: 192.168.1.10
    目标IP: 8.8.8.8

    源MAC: 你的MAC
    目标MAC: 网关MAC  👈 注意这里！
  ```

👉 注意： 💡 IP是远端地址，但MAC是网关！

- step: ⑤ 网关开始“转发”

  网关（路由器）会：
  1. 查路由表
  2. 转发到互联网
  3. 一跳一跳走出去

- step: ⑥ NAT（关键）

  路由器会做： 👉 私网IP → 公网IP转换

  `192.168.1.10 → 你的公网IP`

- step: ⑦ 返回数据

  服务器返回 → 网关 → 再转回你

- ✅ 总结一句话：👉 跨局域网通信 = 发给网关 → 网关帮你转发
